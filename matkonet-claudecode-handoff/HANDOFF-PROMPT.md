# מתכונת · מדריך האש — Prompt פתיחה להמשך ב-Claude Code

הדבק את הקובץ הזה (או הפנה אליו) כשאתה פותח את הפרויקט ב-Claude Code. הוא מסכם את
מצב-הפרויקט, מה נבנה, ומה הצעד הבא.

---

## מה זה הפרויקט

**"מתכונת · מדריך האש" (Matkonet)** — PWA עברית-first (RTL) לבישול-אש: סו-ויד, עישון,
גריל, שרקוטרי. קובץ HTML יחיד (~957KB) שנוצר מ-`build.py`. קטלוג 279 פריטים + מנוע-תזמון +
מזווה + אשף-אירוע + שכבת-AI אופציונלית (BYOK-Gemini, 7 יכולות) + ממשק קולי דו-לשוני.
Local-first (כל הנתונים ב-localStorage). מופץ ב-Netlify (`matkonetesh.netlify.app`, העלאה ידנית).

**בעל הפרויקט:** דודי — חובב-אש מתקדם, עובד בעברית, נייד. **עברית באדיקות בכל מקום.**

---

## כלל-הזהב הארכיטקטוני

**עורכים רק `build.py`, לעולם לא `index.html`** (הוא נוצר). זרימה:
```
python3 build.py    # data.py + קבצי-דאטה → index.html
```
`build.py` (~7400 שורות) מכיל את כל ה-HTML/CSS/JS כמחרוזות-פייתון. תלויות:
`data.py` (CUTS/SPECIALS/GLOSSARY/BUILDS/MAKES), `seasonings.py`, `seasonings_ext.py`,
`seasoning_tags.py`, `house_rub_map.py`, `sausages_new.py`, `descriptions.py`.

**המצב הנוכחי: מהדורה 146.** הפריט `index.html` בחבילה עדכני ל-v146 (reference; Claude Code ייצר מחדש).

## מה נבנה היום (v144 → v146) — הכל shipped

### v144 — פרויקט גדול: 7 שלבים בגרסה אחת
מבוסס על שתי שאלות-עיצוב שאושרו במוקאפים (ראה `mockups/`):

1. **נגישות:** ניגודיות-AA אמיתית (תוקנה מ-2.65:1 שנכשל, ל-5.5:1) + בקרת גודל-טקסט
   (`mk-fontscale`, 4 גדלים, משתנה `--fscale` גלובלי).
2. **ערכות-גוון:** 4 ערכות לבחירה (`mk-theme`) — שמנת-חמה (ברירת-מחדל)/פחם-ולהבה/עץ-ועשן/נחושת-ומלח.
   מנגנון-הערכות הישן היה **מנוטרל בכוונה** בקוד — הופעל ומומש. `THEMES` const + `applyAppearance()`.
3. **פונטים:** 4 זיווגי כותרת+גוף לבחירה (`mk-fontpair`), משתני `--font-display`/`--font-body`.
4. **רמות משתמש:** מתחיל/בינוני/מתקדם (`mk-uilevel`, ברירת-מחדל בינוני=האפליקציה הקיימת) +
   שאלת-אונבורדינג חד-פעמית + בורר ב"🧭 רמת ממשק".
5. **מסלול לפי רמה:** תוכנית-העבודה ב-3 צורות (`mk-tlshape`) — מתחיל→צירים-מתקפלים,
   בינוני→קו-אנכי, מתקדם→צעדים-אופקי. שינוי-ידני נשמר. `renderWorkplanShape()`.
6. **סדר סו-ויד↔עישון:** טוגל per-item (`st.svSmokeOrder`), ברירת-מחדל בטוחה (סו-ויד→עישון עם
   שלב-ייבוש). הסדר ההפוך מוסיף **אזהרת-פסטור** + טמפ׳-עישון-קר מחושבת. `itemStages()` מסתעף לפי סדר.
7. **התאמה לציוד:** תג "🔧 בדוק ציוד" על כרטיסים — מנצל `canSV/canSmoke/canGrill` הקיימים.

**כל ההגדרות ב"עוד → 🎨 מראה / 🧭 רמת ממשק".** הכל בחירת-משתמש חוץ מניגודיות (גבוהה קבועה).

### v145 — 3 באגים מדיווח המשתמש
1. **הגוון הכהה שבור:** שורש אמיתי — `body{background:var(--bg2)}` ו-`var(--card)` היו
   משתנים **שלא הוגדרו בערכות** → טקסט-בהיר על רקע-בהיר. תוקן: `--bg2`/`--card`/`--good`/`--terra-d`
   נוספו לכל 4 הערכות (AA-verified) + `color-scheme` + `-webkit-text-fill-color` נגד dark-mode אוטומטי באנדרואיד.
2. **סדר סו-ויד/עישון לא הופיע בתוכנית-עבודה:** נוסף `tl-orderstrip` לתצוגת plan (מקוצר+מלא).
3. **חקירת זמנים** → הובילה ל-v146.

### v146 — באג-הזמנים, נמצא בסריקה שיטתית
סרקתי את כל 130 הנתחים; **92 עם `svHours=0`** (פירות-ים/ירקות). שוחזר על שרימפס: 4ש ייבוש
לפני 6 דק' צלייה — אבסורד. תוקן: `drySurfaceHours(svHours)` **יחסי** (בריסקט נשאר 4ש, שרימפס→15דק).
+ תיקון-בטיחות: `comboHasSvSmoke` דורש `svHours>=1` (לא להציע סדר-הפוך בלי נתוני-פסטור אמיתיים).

## מוסכמות ובדיקות (חשוב)

- **לכל שינוי:** `python3 build.py` → `node --check` על ה-JS המחולץ → בדיקות → מהדורה חדשה.
- **בדוק התנהגות-נצפית** (DOM/מסך), לא לוגיקה מבודדת. **רגרסיה לכל באג.**
- **P3 בטיחות:** מספרי מלח/ריפוי/פסטור/טמפ׳ תמיד מנתוני-האפליקציה, לעולם לא מ-AI ולא מנוחשים.
- **מפתח היגיינה שנלמד:** אל תצבור בדיקות בהוספה עיוורת (יוצר בלוקים כפולים שדורכים זה על זה דרך
  state גלובלי). ב-Playwright כל spec מבודד עם `beforeEach` שמאפס localStorage.
